SlightSync About 
About

The goal of SlightSync is to be able to watch a presentation from a PDF file together with people at different locations. To do so, the PDF file must be accessible from a web link. Known to work without cross origin problems are PDF files hosted as a shared file on Dropbox.

SlightSync URL options

When you create a new SlightSync URL, the location of the PDF file and the synchronization server to be used are both stored in the URL, which will look something like

https://slightsync.appspot.com/sync.html#id=eyJwZGZVcmwiOiJodHRwczovL2RsLmRyb3Bib3h1c2...

where the id= part is the string that identifies your slides (either master or slave version).

While the synchronization server to use is stored in this identifier, you can still easily override this by appending &server= to the URL, followed by which WebSocket URL you want to use. For example appending &server=wss://slightsync.edm.uhasselt.be/ would use that specific server.

Limits and security

The final URLs to the slides are just difficult URLs. They encode the information needed to view the PDF file in a synchronized way (location of the PDF and synchronization server options) and should be quite difficult to guess. And this is the only security precaution there is, there are no logins or passwords involved, so don't give the URLs to someone that should not see the slides.

The URLs are not stored anywhere, so if you lose them, they're lost forever. But a new set of URLs can easily be created again of course.

JavaScript, Python, CSS used

Here's a list of things I used to make this application:

  • The nice dialog boxes that appear from time to time: modalbox
  • Needed by 'modalbox' and for some other effects: scriptaculous and prototype
  • Additional dialog boxes: Vex
  • Needed by the previous plugin and for some general scripting: jQuery
  • To render PDF files in the browser: pdf.js
  • Create QR tags for URLs: qrcode.js
  • Detect gestures on mobile devices: Hammer.js
  • CSS inspiration for the headers on the page comes from this page: see example two
  • Nice CSS input boxes: using the 'Glow' style
  • For displaying the screenshots on the main page: Lightbox
